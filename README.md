# TRPG 跑团助手 (Web版)

这是一个基于 Web 的多人在线 TRPG (Tabletop Role-Playing Game) 跑团平台，旨在提供沉浸式的跑团体验。

## 🛠️ 技术栈

- **前端**: React 18, TypeScript, Tailwind CSS, Framer Motion, Lucide React
- **后端**: Node.js, Express, Socket.io
- **构建工具**: Vite
- **运行环境**: Node.js (推荐 v18+)

## 🚀 本地部署指南

### 1. 环境准备
确保您的电脑上已安装 [Node.js](https://nodejs.org/) (建议版本 v18 或更高)。

### 2. 获取代码
将项目文件下载或克隆到本地目录。

### 3. 安装依赖
在项目根目录下打开终端，运行以下命令安装所需依赖：

```bash
npm install
```

### 4. 启动开发服务器
运行以下命令启动全栈开发服务器（包含前端和后端）：

```bash
npm run dev
```

### 5. 访问应用
服务器启动成功后，打开浏览器访问：
`http://localhost:3000`

> **注意**: 本地开发时，多开浏览器窗口即可模拟多玩家联机测试。

## 💡 交互设计思路

本项目的设计核心是**“沉浸感”**与**“实时性”**，力求在网页端还原线下跑团的氛围。

### 1. 视觉风格 (Visual Identity)
*   **暗黑美学**: 采用 Zinc-950/Black 作为主色调，搭配 Amber (琥珀色) 作为强调色，营造神秘、复古的 TRPG 氛围。
*   **界面布局**: 
    *   **左侧 (Character)**: 玩家自身的角色卡，实时显示 HP、SAN 值及核心属性，时刻提醒玩家当前状态。
    *   **中间 (Stage)**: 游戏的“主舞台”。用于展示剧情文本、场景描述以及掷骰子动画。采用流式文本打字机效果，增强叙事感。
    *   **右侧 (Utility)**: 功能聚合区。包含系统菜单、队伍状态查看以及 OOC (Out of Character) 闲聊频道。

### 2. 核心交互流程 (User Flow)

#### A. 登录与大厅 (Lobby)
*   **极简入口**: 仅需输入昵称即可开始。
*   **房间系统**: 支持创建新房间或通过 6 位房间号加入现有房间。
*   **等待大厅**: 
    *   可视化展示已加入的玩家。
    *   提供实时聊天功能，方便玩家在开局前沟通（如“车卡”讨论）。
    *   房主控制游戏开始权限。

#### B. 游戏进行时 (In-Game)
*   **实时同步**: 所有玩家的操作、掷骰结果、剧情推进通过 Socket.io 毫秒级同步。
*   **指令系统**: 
    *   支持自然语言输入行动（如“我尝试侦查这个房间”）。
    *   支持 `/r` 指令进行掷骰检定。
*   **动态反馈**: 
    *   **队伍面板**: 点击右侧菜单的“队伍”按钮，可查看队友的职业、等级及当前生命/理智状态，方便辅助决策（如牧师决定治疗对象）。
    *   **菜单系统**: 折叠式设计，将物品栏、技能书、任务记录等次要信息收纳，保持主界面整洁。

### 3. 特色功能
*   **全屏自适应**: 移除了最大宽度限制，利用现代宽屏优势，提供更广阔的视觉体验。
*   **平滑动画**: 使用 Framer Motion 实现界面切换、列表加载和骰子滚动的平滑过渡。
*   **即时通讯**: 内置聊天面板，区分游戏内指令与游戏外闲聊。

## 📝 开发备注
*   **Socket 服务**: 后端逻辑位于 `server.ts`，处理房间管理 (`socketService.ts` 对应前端逻辑)。
*   **数据结构**: 核心类型定义在 `src/types/GameData.ts`。
